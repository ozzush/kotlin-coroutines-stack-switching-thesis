\section{WebAssembly and Stack-switching}
\label{sec:webassembly}

Stack-switching proposal introduces a type for continuation and the corresponding instructions to work with it. Continuation is an object that represents a function at some point in its execution. That point is either the beginning of the function or a marked suspension point. Resuming continuation resumes the function from the recorded point. Continuation can also be intuitively understood as "the rest of a computation."

A good intuition for understanding resuming and suspending in the stack-switching proposal is that it is similar to calling a function inside a try-catch block. The function can suspend by throwing different types of exceptions (suspension tags) and the try-catch block can provide different handlers for the corresponding suspension tags.

In Figure~\ref{fig:stack-switching-cfg} is a simplified example in Wasm that showcases the basics of using continuations.

\begin{figure}[H]
\centering
\begin{minipage}[t]{0.48\textwidth}
\begin{lstlisting}[language=wasm, label={lst:wasm-example}, basicstyle=\tiny\ttfamily]
(func $generator
  ...
  (loop $loop
    ;; Suspend execution, pass current value of $i to consumer.
    (suspend $generate (local.get $i))
    ...
  )
)

(func $consumer
 ...
 ;; Create continuation executing function $generator.
 (local.set $generator_cont (cont.new $cont_type (ref.func $generator)))

 (loop $loop
   (block $on_generate (result i32 (ref $cont_type))
     ;; Resume continuation $generator_cont.
     ;; When $generate tag is thrown,
     ;; go to label $on_generate (points to right after this block)
     (resume $cont_type (on $generate $on_generate) (local.get $generator_cont))
     ;; $generator returned: no more data.
     (return)
   )
   ;; $generator suspended.
   ;; The stack now contains the value
   ;; produced by the generator and the new continuation.
   ;; Save continuation to resume it in the next iteration
   (local.set $generator_cont)
   ...
 )
)
\end{lstlisting}
\end{minipage}%

\section{Kotlin Coroutines}
\label{sec:kotlin-coroutines}

Kotlin provides several low-level intrinsics and primitives for implementing coroutines. These primitives form the foundation for higher-level concurrency abstractions such as the kotlinx.coroutines library. The key primitives are:

\subsection*{Continuation}

\texttt{Continuation<T>} is an interface that represents a continuation, similar to the continuation concept in WebAssembly described earlier. It represents a coroutine at a suspension point - a place where the coroutine has paused its execution. The interface has two main components:

\begin{itemize}
    \item \texttt{context: CoroutineContext} - the context of the coroutine
    \item \texttt{resumeWith(result: Result<T>)} - resumes the coroutine execution, passing a result value that becomes the return value of the suspension point
\end{itemize}

\subsection*{startCoroutineUninterceptedOrReturn}

This intrinsic function starts a coroutine. It is defined as an extension function on suspending function types:

\begin{lstlisting}[language=Kotlin, basicstyle=\small\ttfamily]
(suspend () -> T).startCoroutineUninterceptedOrReturn(
    completion: Continuation<T>
): Any?
\end{lstlisting}

The function immediately starts the suspending function with the given completion continuation. It returns:
\begin{itemize}
    \item \texttt{COROUTINE\_SUSPENDED} if the coroutine was suspended
    \item The result value if the coroutine completed immediately without suspension
\end{itemize}

This primitive is used when you need direct control over coroutine execution.

\subsection*{createCoroutineUnintercepted}

This intrinsic creates a coroutine. It is defined as an extension function on suspending function types:

\begin{lstlisting}[language=Kotlin, basicstyle=\small\ttfamily]
(suspend () -> T).createCoroutineUnintercepted(
    completion: Continuation<T>
): Continuation<Unit>
\end{lstlisting}

Unlike \texttt{startCoroutineUninterceptedOrReturn}, this function does not start the coroutine immediately. Instead, it returns a \texttt{Continuation<Unit>} that can be resumed later to start the coroutine execution. This allows you to control when the coroutine begins.

\subsection*{suspendCoroutineUninterceptedOrReturn}

This intrinsic function gives you access to the current continuation inside a suspend function. It allows the function to control whether it suspends or returns immediately:

\begin{lstlisting}[language=Kotlin, basicstyle=\small\ttfamily]
suspendCoroutineUninterceptedOrReturn<T>(
    block: (Continuation<T>) -> Any?): T
\end{lstlisting}

The \texttt{block} receives the current continuation and can:
\begin{itemize}
    \item Return \texttt{COROUTINE\_SUSPENDED} to indicate that the coroutine has suspended and will be resumed later
    \item Return a value directly to complete immediately without suspension
\end{itemize}

This is the most basic building block for implementing custom suspending functions, as it gives you direct access to the continuation and control over suspension behavior.
